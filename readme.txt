2 四则运算
2.1技术要求：
2.1.1.前后端分离
2.1.2.前端基于VUE、Element UI实现
2.1.3.后端使用Spring Boot、Spring MVC 、MyBatis
2.1.4.基于RESTful风格。
2.1.5.四则运算公式内的参数替换及计算在后端（服务端）完成。

2.2功能描述：
2.2.1主界面：文本域输入框，比如输入公式  (${参数1} + ${参数2} + ${参数3} - ${参数4})/(${参数5} + ${参数6})

2.2.2.页面输入区，可输入带有参数的四则运算公式，参数格式为：参数开头为美元符号，之后是大括号内的参数名，如上图。

2.2.3.点击“验证公式”按钮，会弹出公式验证对话框，显示如下图。

2.2.4.“公式验证对话框”会根据公式中参数的个数，动态生成参数输入框。例如，如果有6个参数，公式验证对话框就有6个输入框，每个输入框的前的Label为参数名。

2.2.5.在输入框中输入数字后，点击“运行”按钮，会将四则运算公式和所有参数值共同发送到服务端。

2.2.6.在服务端实现将四则运算公式中的参数替换为输入的值，并执行四则运算。

2.2.7.将运算返回给前端，显示在Label“运行结果：”之后。

2.2.8.如果输入参数时，输入非数字或未输入任何数字，点击“运行”按钮时则提示参数错误。
